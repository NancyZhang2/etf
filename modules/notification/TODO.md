# Notification模块 — 任务队列

> Orchestrator 监控此文件，发现"待开始"任务时启动 Worker 执行。
> 状态：待开始 → 进行中 → 已完成 / BLOCKED

---

## N1. 微信公众号模板消息推送
状态：待开始
优先级：P0
描述：接入微信公众号模板消息 API，实现推送服务。推送内容：每日交易信号（BUY/SELL明细）、重要研报摘要。推送时间：交易日16:30。需实现：access_token管理、模板消息发送、发送失败重试。
验收：可向关注用户推送模板消息。

## N2. SMTP 邮件推送
状态：待开始
优先级：P0
描述：实现 HTML 格式邮件推送。内容类型：每日交易信号（含表格）、策略表现周报（内嵌净值曲线图表）、重要研报摘要。支持 SMTP 配置（host/port/user/password）。
验收：可发送HTML格式邮件，图表正常显示。

## N3. 订阅管理服务
状态：待开始
优先级：P0
描述：实现用户订阅 CRUD：按策略订阅、按ETF订阅、按类型订阅（信号/研报/周报）。频率控制：实时/每日汇总/每周汇总。渠道选择：微信/邮件/两者。写入 user_subscriptions 表。
验收：订阅增删改查正常工作。

## N4. 推送调度与内容组装
状态：待开始
优先级：P0
依赖：N1, N2, N3
描述：每日16:30（数据更新+信号生成完成后）：根据用户订阅配置，组装推送内容，通过对应渠道发送。支持每日汇总模式（一封邮件/一条消息包含当日全部信号）。
验收：推送可自动触发，内容正确组装。

## N5. Notification 模块 REST API
状态：待开始
优先级：P0
描述：实现 CLAUDE.md 中定义的4个 Notification API 端点：POST /subscribe、DELETE /subscribe/{id}、GET /subscriptions、PUT /subscribe/{id}。统一响应格式。
验收：所有端点可调用。

## N6. 策略表现周报
状态：待开始
优先级：P1
依赖：N2
描述：每周日生成策略表现周报：本周信号汇总、虚拟持仓收益、各策略本周表现排名。通过邮件推送（HTML+图表）。
验收：周报可自动生成并推送。
