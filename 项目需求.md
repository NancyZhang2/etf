## 一、产品定位

### 1.1 一句话定义

面向有投资经验的个人投资者的 **全A股场内ETF量化投研平台**，
集数据自动更新、策略编写与回测、研报AI分析、每日交易信号推送于一体。

### 1.2 目标用户画像

| 特征 | 描述 |
|------|------|
| 投资经验 | 2年以上A股投资经验，了解ETF基本概念 |
| 技术能力 | 不要求编程能力，但能理解参数配置和指标含义 |
| 使用场景 | 日常：手机查看信号和研报；周末：PC端调整策略和看回测 |
| 核心诉求 | 省时间（自动数据+自动信号）、有依据（回测数据+研报分析）、可验证（实盘跟踪） |
| 痛点 | 手动跟踪多只ETF耗时、策略凭感觉缺数据支撑、研报散落难整合 |

### 1.3 竞品差异化

| 维度 | 果仁网/聚宽/BigQuant | 韭圈儿/理杏仁 | ETFreplay | **我们** |
|------|----------------------|---------------|-----------|---------|
| 标的范围 | 全品类（股票/期货/ETF） | 基金/指数 | 美国ETF | **专精A股场内ETF** |
| 策略来源 | 用户自建 | 无策略 | 用户配置 | **内置经典策略 + 竞品逆向策略** |
| 研报能力 | 无 | 无 | 无 | **AI自动研报分析** |
| 信号服务 | 工具型，用户自取 | 无 | 无 | **每日主动推送（微信+邮件）** |
| 数据更新 | 手动或半自动 | 依赖第三方 | 自动 | **15:30自动增量更新** |
| 使用门槛 | 需编程或学习平台 | 低 | 低 | **零编程，参数化配置** |

## 三、模块一：数据引擎（Data）

### 3.1 功能需求

#### F-DATA-01：ETF基础信息管理

| 项目 | 说明 |
|------|------|
| 描述 | 维护全A股场内ETF基础信息库 |
| 数据来源 | akshare `fund_etf_spot_em()` 或等效接口 |
| 覆盖范围 | 上交所 + 深交所所有场内ETF（预计900+只） |
| 存储字段 | 代码、名称、分类（宽基/行业/主题/商品/债券/货币/跨境）、交易所 |
| 更新频率 | 每周一次全量同步（新上市/退市ETF） |
| 分类规则 | 按底层资产自动分类，支持人工修正 |

#### F-DATA-02：历史行情一次性拉取

| 项目 | 说明 |
|------|------|
| 描述 | 拉取所有ETF自上市以来的全部日K线数据 |
| 数据字段 | 日期、开盘价、最高价、最低价、收盘价、成交量、成交额 |
| 数据来源 | akshare `fund_etf_hist_em()` |
| 容错机制 | 单只ETF拉取失败自动重试2次，失败记录到日志，不阻塞其他ETF |
| 预估数据量 | 约900只ETF × 平均2000交易日 = 180万条记录 |

#### F-DATA-03：每日增量更新

| 项目 | 说明 |
|------|------|
| 描述 | 每个交易日收盘后自动拉取当日全部ETF行情 |
| 触发时机 | 每个交易日15:30由定时任务触发 |
| 更新逻辑 | 仅拉取数据库中尚无记录的最新交易日数据 |
| 交易日判断 | 跳过周末和法定节假日（基于akshare交易日历） |
| 数据校验 | 更新后自动校验：记录数 ≥ 已知ETF数量的90%，否则告警 |

#### F-DATA-04：数据清洗

| 项目 | 说明 |
|------|------|
| 缺失值处理 | 停牌日不补数据，标记为停牌状态 |
| 异常值检测 | 单日涨跌幅超过±15%的记录标记审核（ETF极端情况） |
| 复权处理 | 存储前复权价格，同时保留原始价格字段 |
| 去重校验 | (code, trade_date) 唯一约束，重复数据自动跳过 |

#### F-DATA-05：数据服务API

对外REST API，供Quant/Research/Frontend模块调用：

```
GET  /api/v1/etf/list                    # ETF列表（支持分类筛选）
GET  /api/v1/etf/list/categories         # 获取所有分类
GET  /api/v1/etf/{code}/info             # 单只ETF详情
GET  /api/v1/etf/{code}/daily            # 日K线（支持日期范围）
GET  /api/v1/etf/{code}/latest           # 最新行情
GET  /api/v1/etf/batch/daily             # 批量获取多只ETF行情
POST /api/v1/data/sync                   # 触发增量更新
GET  /api/v1/data/status                 # 数据更新状态
GET  /api/v1/data/calendar               # 交易日历
```

## 四、模块二：策略引擎（Quant）

### 4.1 策略体系设计

策略分为两大类、六个子类：

```
策略体系
├── A. 经典量化策略（学术/业界公认有效的ETF策略）
│   ├── A1. 动量轮动策略
│   ├── A2. 均线趋势策略
│   ├── A3. 网格交易策略
│   └── A4. 大类资产配置策略
│
└── B. 竞品逆向工程策略（从竞品平台分析提取的策略逻辑）
    ├── B1. 蛋卷二八轮动复刻版
    └── B2. 果仁行业轮动复刻版
```

### 4.2 经典量化策略详细需求

#### A1. 动量轮动策略

**策略逻辑**：
在ETF池中，按过去N日收益率（动量值）排名，买入排名前K只ETF，等权或动量加权持有。
定期（每周/每月）重新排名并调仓。若所有ETF动量均为负值，转为持有货币基金ETF（避险）。

**可配置参数**：

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| etf_pool | 用户选择 | 全部A股ETF | 轮动标的池 |
| lookback_days | 20 | 5~120 | 动量回看天数 |
| hold_count | 3 | 1~10 | 持仓ETF数量 |
| rebalance_period | weekly | daily/weekly/monthly | 调仓周期 |
| weight_method | equal | equal/momentum | 持仓权重方式 |
| use_hedge | true | true/false | 是否启用避险机制 |
| hedge_etf | 511880 | 货币ETF代码 | 避险标的 |
| hedge_threshold | 0 | -0.1~0 | 全负时转避险的阈值 |

**信号输出**：每个调仓日输出 BUY/SELL/HOLD 信号及对应ETF代码、目标权重、调仓理由。

**参考来源**：
- 蛋卷基金二八轮动策略核心逻辑（20日动量对比）
- 集思录ETF轮动实盘（20日ROC选前三名）
- 华泰证券"时序动量+风险预算"框架

---

#### A2. 均线趋势策略

**策略逻辑**：
基于移动均线判断ETF趋势方向。价格在均线之上为上升趋势（持有），跌破均线为下降趋势（卖出转避险）。
支持单均线和双均线交叉两种模式。

**单均线模式**：
- 收盘价 > SMA(N) → BUY/HOLD
- 收盘价 < SMA(N) → SELL，转避险标的

**双均线模式**：
- SMA(短期) 上穿 SMA(长期) → 金叉买入
- SMA(短期) 下穿 SMA(长期) → 死叉卖出

**可配置参数**：

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| target_etf | 用户选择 | 单只ETF | 交易标的 |
| ma_type | SMA | SMA/EMA | 均线类型 |
| ma_mode | single | single/dual | 单均线/双均线 |
| ma_period | 20 | 5~250 | 均线周期（单均线模式） |
| short_period | 5 | 3~60 | 短期均线（双均线模式） |
| long_period | 20 | 10~250 | 长期均线（双均线模式） |
| hedge_etf | 511880 | 货币ETF代码 | 避险标的 |

**增强选项**：
- 大盘过滤器：沪深300低于SMA(120)时强制避险，不管个别ETF信号
- 均线能量指标：衡量均线累计上涨/下跌期数，判断趋势强度

**参考来源**：
- ETFreplay的Moving Average Backtest模块
- "均线能量"改良动量策略（2013-2025年化31.3%）

---

#### A3. 网格交易策略

**策略逻辑**：
在设定的价格区间内按等距（或等比）网格划分，每触碰一个格线执行买入或卖出。
适合震荡行情（A股约80%时间为震荡市），天然具备"低买高卖"属性。

**可配置参数**：

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| target_etf | 用户选择 | 单只ETF | 交易标的 |
| grid_type | arithmetic | arithmetic/geometric | 等差/等比网格 |
| price_upper | 自动计算 | > 当前价 | 网格上界 |
| price_lower | 自动计算 | < 当前价 | 网格下界 |
| grid_count | 10 | 5~50 | 网格数量 |
| order_amount | 1000 | > 0 | 每格交易金额(元) |
| total_budget | 100000 | > 0 | 总投入资金(元) |

**信号特点**：
- 非定期信号，而是盘中价格触发
- 回测时按日K线模拟（使用最高价/最低价判断是否触格）
- 信号带有"第N格买入/卖出"的具体标注

**参考来源**：
- 华宝智投"ETF网格机器人"产品逻辑
- 量化复合策略：ETF底仓 + 网格交易 + 动量止盈

---

#### A4. 大类资产配置策略

**策略逻辑**：
选取低相关性的跨类别ETF（股票、债券、黄金、商品、海外），按配置模型分配权重，定期再平衡。
内置三种经典配置模型。

**模型一：全天候策略（桥水 All Weather）**

| 资产类别 | ETF示例 | 权重 |
|----------|---------|------|
| A股宽基 | 沪深300ETF (510300) | 30% |
| 长期国债 | 国债ETF (511010) | 40% |
| 中期国债 | 5年国债ETF (511020) | 15% |
| 黄金 | 黄金ETF (518880) | 7.5% |
| 商品 | 豆粕ETF (159985) | 7.5% |

**模型二：风险平价策略（Risk Parity）**

根据各资产历史波动率动态计算权重，使每类资产对组合风险贡献相等。
每月重新计算权重并调仓。

**模型三：股债动态平衡**

根据市场估值（沪深300 PE百分位）动态调整股债比例：
- PE百分位 < 30%（低估）：股票70% + 债券30%
- PE百分位 30%~70%（正常）：股票50% + 债券50%
- PE百分位 > 70%（高估）：股票30% + 债券70%

**可配置参数**：

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| model | all_weather | all_weather/risk_parity/dynamic_balance | 配置模型 |
| etf_mapping | 内置默认 | 用户可替换 | 各资产类别对应ETF |
| rebalance_period | monthly | weekly/monthly/quarterly | 再平衡周期 |
| deviation_threshold | 5% | 1%~20% | 偏离阈值触发再平衡 |

**参考来源**：
- Portfolio Visualizer 因子分析和Tactical Allocation模型
- BigQuant 风险平价ETF配置策略
- 桥水基金全天候策略

### 4.3 竞品逆向工程策略详细需求

#### B1. 蛋卷二八轮动复刻版

**原版逻辑复刻**：

```
每个交易日收盘后：
  计算沪深300指数 20日涨幅 = (今日收盘 - 20日前收盘) / 20日前收盘
  计算中证500指数 20日涨幅 = (今日收盘 - 20日前收盘) / 20日前收盘

  IF 沪深300涨幅 > 0 AND 沪深300涨幅 >= 中证500涨幅:
      → 持有沪深300ETF
  ELIF 中证500涨幅 > 0 AND 中证500涨幅 > 沪深300涨幅:
      → 持有中证500ETF
  ELSE (两者均下跌):
      → 持有货币基金ETF
```

**增强优化点**（区别于原版）：
1. 可配置回看天数（原版固定20日，支持10/20/60日对比）
2. 可替换标的对（原版固定300/500，支持换成其他宽基ETF对）
3. 增加切换缓冲：涨幅差距 < 1%时不切换，减少频繁调仓
4. 增加大盘保护：若沪深300低于250日均线，强制避险

**可配置参数**：

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| large_cap_etf | 510300 | ETF代码 | 大盘标的 |
| small_cap_etf | 510500 | ETF代码 | 小盘标的 |
| hedge_etf | 511880 | ETF代码 | 避险标的 |
| lookback_days | 20 | 10/20/60 | 回看天数 |
| switch_buffer | 0.01 | 0~0.05 | 切换缓冲（涨幅差阈值） |
| use_ma_protection | true | true/false | 是否启用均线保护 |
| ma_protection_period | 250 | 120/250 | 保护均线周期 |

**参考来源**：
- 蛋卷斗牛二八轮动策略官方规则
- 果仁网社区对二八轮动的回测分析

#### B2. 果仁行业轮动复刻版

**原版逻辑复刻**：

```
ETF池：选取主流行业ETF（银行、证券、军工、医药、消费、科技、新能源等）

每个调仓日（每周五收盘）：
  FOR 每只ETF in ETF池:
      计算 20日净值增长排名（升序排名，涨幅越大排名越前）
      计算 溢价率排名（升序排名，溢价越低排名越前）
      综合排名 = 0.7 × 动量排名 + 0.3 × 溢价率排名

  选取综合排名前N只 → 等权买入持有至下个调仓日
  卖出不在前N名的持仓
```

**增强优化点**：
1. 可配置因子权重（动量/溢价/波动率/成交量）
2. 可配置行业ETF池（从平台ETF列表中按分类选取）
3. 增加行业集中度限制：同一大类行业最多持有2只ETF
4. 增加回撤控制：持仓ETF回撤超过阈值强制卖出

**可配置参数**：

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| etf_pool | 行业ETF列表 | 用户选择 | 轮动标的池 |
| hold_count | 3 | 1~10 | 持仓数量 |
| rebalance_day | friday | 周几 | 调仓日 |
| momentum_weight | 0.7 | 0~1 | 动量因子权重 |
| premium_weight | 0.3 | 0~1 | 溢价率因子权重 |
| lookback_days | 20 | 5~60 | 动量回看天数 |
| max_same_sector | 2 | 1~5 | 同行业最大持仓数 |
| drawdown_stop | 0.15 | 0.05~0.30 | 回撤止损阈值 |

**参考来源**：
- 果仁网行业ETF轮动策略（年化37%回测）
- 集思录20日净值增长+溢价率双因子轮动实盘

### 4.4 回测引擎需求

#### F-QUANT-01：回测核心功能

| 项目 | 说明 |
|------|------|
| 引擎 | vectorbt（禁止使用backtrader） |
| 回测区间 | 支持自定义起止日期，默认近5年 |
| 逐年回测 | 自动输出近5年每年的独立回测结果 + 全区间结果 |
| 交易成本 | 默认ETF佣金万1，支持自定义 |
| 滑点模拟 | 默认0.1%，支持自定义 |
| 初始资金 | 默认100万，支持自定义 |
| 基准对比 | 默认沪深300，支持选择其他ETF作为基准 |

#### F-QUANT-02：回测结果指标

每次回测输出以下指标：

**收益指标**：
- 总收益率
- 年化收益率（CAGR）
- 月均收益率
- 单日最大收益 / 单日最大亏损

**风险指标**：
- 最大回撤（金额 + 百分比 + 持续天数 + 恢复天数）
- 年化波动率
- 下行波动率

**风险调整指标**：
- 夏普比率（Sharpe Ratio）
- 索提诺比率（Sortino Ratio）
- 卡尔玛比率（Calmar Ratio）

**交易指标**：
- 总交易次数
- 胜率（盈利交易占比）
- 盈亏比
- 平均持仓天数
- 换手率

#### F-QUANT-03：回测可视化

| 图表 | 说明 |
|------|------|
| 净值曲线 | 策略净值 vs 基准净值，双线对比 |
| 回撤曲线 | 回撤幅度随时间变化 |
| 月度收益热力图 | 行 = 年份，列 = 月份，颜色 = 收益率 |
| 年度收益柱状图 | 逐年收益率 vs 基准收益率 |
| 持仓变动图 | 时间轴上的买入卖出标记 |
| 收益分布直方图 | 日/月收益率频率分布 |

#### F-QUANT-04：策略参数优化

| 项目 | 说明 |
|------|------|
| 参数扫描 | 支持对任意1~2个参数进行网格搜索 |
| 结果展示 | 热力图（双参数）或折线图（单参数）展示不同参数下的夏普比率 |
| 过拟合警告 | 当最优参数与邻近参数表现差异过大时，提示可能过拟合 |
| Walk-Forward | 支持滚动窗口前向测试，验证参数稳定性 |

### 4.5 信号生成需求

#### F-QUANT-05：每日信号生成

| 项目 | 说明 |
|------|------|
| 触发时机 | 每个交易日数据更新完成后自动运行 |
| 运行范围 | 所有已启用的策略 |
