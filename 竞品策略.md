## B1. 蛋卷二八轮动

### 来源
蛋卷基金（雪球旗下），策略代码 CSI001TH。
由天弘基金提供基础策略，蛋卷开发。

### 原版逻辑

```
每个交易日T收盘后：

  沪深300涨幅 = (沪深300今日收盘 - 沪深300第T-20日收盘) / 沪深300第T-20日收盘
  中证500涨幅 = (中证500今日收盘 - 中证500第T-20日收盘) / 中证500第T-20日收盘

  IF 沪深300涨幅 > 0 AND 沪深300涨幅 >= 中证500涨幅:
      次日持有 → 沪深300指数基金
  ELIF 中证500涨幅 > 0 AND 中证500涨幅 > 沪深300涨幅:
      次日持有 → 中证500指数基金
  ELSE (两者均为负):
      次日持有 → 货币基金（原版用国债指数）

  若当前已持有目标基金 → 不操作
```

### 关键细节
- **动量窗口**：固定20个交易日
- **执行时点**：T日收盘计算，T+1日收盘价执行
- **份额可用**：T+2日份额到账，到账前不能再次调仓
- **赎回费保护**：持仓不足7天且7天内连续两次触发1.5%赎回费时，取消第二次调仓
- **原版标的**：天弘沪深300 / 天弘中证500 / 天弘弘运宝货币A

### 原版回测数据
- 2006-2016十年回测：涨幅约32倍，年化约41%（不含手续费）
- 2008年大熊市：显著跑赢大盘
- 2010-2013年震荡市：连续四年小幅亏损，但跌幅小于大盘
- **核心缺陷**：交易成本若无法控制在1%以内，基本无超额收益

### 我们的复刻方案

**ETF化改造**（原版用场外基金，我们用场内ETF）：

| 原版标的 | 我们的标的 | 优势 |
|----------|-----------|------|
| 天弘沪深300基金 | 沪深300ETF (510300) | 佣金万1，T+1可卖 |
| 天弘中证500基金 | 中证500ETF (510500) | 无赎回费问题 |
| 天弘货币基金 | 银华日利 (511880) | 场内货币，即时可用 |

**增强优化**：

| 优化点 | 原版 | 我们的增强 | 原因 |
|--------|------|-----------|------|
| 动量窗口 | 固定20日 | 可配置10/20/60日 | 不同市场周期适用不同窗口 |
| 标的对 | 固定300/500 | 可替换任意两只宽基ETF | 扩展策略适用性 |
| 切换缓冲 | 无 | 涨幅差 < 1%时不切换 | 减少震荡市无效调仓 |
| 大盘保护 | 无 | 沪深300 < 250日均线时强制避险 | 熊市初期提前止损 |
| 赎回费 | 需特殊处理 | ETF场内交易无此问题 | 场内ETF天然优势 |

**可配置参数**：

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| large_cap_etf | 510300 | ETF代码 | 大盘标的 |
| small_cap_etf | 510500 | ETF代码 | 小盘标的 |
| hedge_etf | 511880 | ETF代码 | 避险标的 |
| lookback_days | 20 | 10/20/60 | 动量回看天数 |
| switch_buffer | 0.01 | 0~0.05 | 切换缓冲阈值 |
| use_ma_protection | true | true/false | 均线保护开关 |
| ma_protection_period | 250 | 120/250 | 保护均线周期 |

## B2. 蛋卷二八轮动Plus（钝化增强版）

### 来源
蛋卷基金，在B1基础上优化持有体验。

### 原版逻辑

在B1基础上增加**钝化机制**：

```
在B1的判断逻辑之上增加：

  # 钝化条件：趋势判断更严格
  IF 信号方向与当前持仓相同:
      不操作（维持持仓）
  ELIF 信号方向发生变化:
      检查信号是否连续N天一致（钝化窗口）
      IF 连续N天信号一致:
          执行调仓
      ELSE:
          不操作（等待信号确认）
```

### 核心改进
- **减少震荡市频繁调仓**：信号必须持续确认才执行
- **趋势判断更严**：避免单日噪音触发调仓
- **改善持有体验**：降低震荡市的"打脸"频率

### 我们的复刻方案

在B1策略基础上增加钝化参数：

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| use_blunting | true | true/false | 是否启用钝化 |
| blunting_days | 3 | 2~5 | 信号需连续确认的天数 |
| （继承B1全部参数） | | | |

## B3. 蛋卷八仙过海

### 来源
蛋卷基金，由招商基金提供基础策略。

### 原版逻辑

```
第一层：大盘择时（决定是否入场）
  计算上证50指数的修正40日均线
  IF 上证50日K线 上穿 修正40日均线 且次日站稳:
      → 入场信号（从债基转入股基）
  ELIF 上证50日K线 下穿 修正40日均线:
      → 离场信号（从股基转入债基）

第二层：行业选择（入场后买什么）
  ETF池 = 8个行业指数基金
  计算每只基金近10个交易日涨幅
  选取涨幅最大的前3只 → 等权买入

调仓周期：
  择时信号变化时调仓（非固定周期）
  入场状态下，每10个交易日重新排名行业
```

### 核心特点
- **两层决策**：先判大势（上证50均线），再选行业（10日动量Top3）
- **换仓频率低于二八轮动**：只在均线交叉时大调，日常只换行业
- **8个行业**：银行、证券、军工、医药、消费、科技、新能源、地产（各显神通）

### 我们的复刻方案

```
第一层：大盘择时
  计算沪深300（替代上证50）的SMA(40)
  IF close > SMA(40) AND 前一日close <= SMA(40):
      → 入场
  ELIF close < SMA(40):
      → 离场，全仓转货币ETF

第二层：行业选择
  行业ETF池 = [银行512800, 证券512880, 军工512660,
               医药512010, 消费159928, 半导体512480,
               新能源515030, 光伏515790]
  计算每只ETF近N日涨幅
  选取Top K只 → 等权买入
```

**可配置参数**：

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| timing_index | 510300 | ETF代码 | 择时基准（沪深300ETF） |
| timing_ma_period | 40 | 20~60 | 择时均线周期 |
| sector_etf_pool | 8只行业ETF | 用户选择 | 行业标的池 |
| momentum_days | 10 | 5~20 | 行业动量回看天数 |
| hold_count | 3 | 1~5 | 持有行业数量 |
| hedge_etf | 511880 | ETF代码 | 离场避险标的 |
| sector_rebalance_days | 10 | 5~20 | 行业重排周期（交易日） |

## B4. 蛋卷安睡二八平衡

### 来源
蛋卷基金，经典股债再平衡策略。

### 原版逻辑

```
初始配置：
  债券基金 = 80%
  沪深300指数基金 = 20%

再平衡触发：
  每月15日检查持仓比例
  IF 沪深300基金市值占比 < 15% OR > 25%:
      执行再平衡，恢复至 80/20 初始比例
  ELSE:
      不操作
```

### 核心特点
- **极简规则**：固定比例 + 偏离阈值触发再平衡
- **低频调仓**：每月最多检查一次
- **防守型策略**：80%债券，适合风险厌恶型投资者
- **利用均值回归**：股票涨多了自动卖、跌多了自动买

### 我们的复刻方案

```
初始配置：
  bond_etf (国债ETF 511010) = bond_ratio (80%)
  stock_etf (沪深300ETF 510300) = 1 - bond_ratio (20%)

每月check_day检查：
  stock_weight = stock_etf市值 / 总市值
  IF abs(stock_weight - target_stock_ratio) > deviation_threshold:
      再平衡至目标比例
```

**可配置参数**：

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| stock_etf | 510300 | ETF代码 | 股票端标的 |
| bond_etf | 511010 | ETF代码 | 债券端标的 |
| bond_ratio | 0.80 | 0.5~0.9 | 债券占比 |
| check_day | 15 | 1~28 | 每月检查日 |
| deviation_threshold | 0.05 | 0.03~0.10 | 偏离阈值（5%触发） |

## B6. 果仁动量溢价双因子轮动

### 来源
果仁网社区 + 集思录实盘验证。年化37%回测。

### 原版逻辑

```
ETF池：选取流动性好的ETF/LOF/封基（约14只）

每个调仓日（每周五收盘）：
  FOR 每只标的 in 标的池:
      factor_1 = 20日净值增长率       # 动量因子
      factor_2 = 溢价率（越低越好）    # 估值因子

      momentum_rank = 按factor_1降序排名  # 涨得多排名靠前
      premium_rank  = 按factor_2升序排名  # 溢价低排名靠前

      composite_score = 0.7 × momentum_rank + 0.3 × premium_rank

  选取composite_score最优的前N只 → 等权买入
  卖出不在前N名的持仓

额外过滤（增强版）：
  IF 标的20日增长率 < 0:
      排除出候选池（不买下跌标的）
  IF 标的20日均成交额 < 10万:
      排除（流动性过滤）
```

### 核心特点
- **双因子排名**：动量（强势追涨）+ 溢价率（便宜优先）
- **周频调仓**：每周五执行，平衡及时性和交易成本
- **流动性过滤**：成交额门槛排除流动性差的标的
- **负动量过滤**：下跌标的不入选，变相实现择时

### 我们的复刻方案

```
默认ETF池（行业ETF为主）：
  [512800银行, 512880证券, 512660军工, 512010医药,
   159928消费, 515030新能源, 512480半导体, 515790光伏,
   512690酒, 159869游戏]

每个rebalance_day收盘后：
  FOR etf in pool:
      momentum = (close - close_N_days_ago) / close_N_days_ago
      premium = 溢价率（如有IOPV数据）或用0替代

      IF momentum < momentum_threshold:
          排除
      IF avg_amount_20d < min_amount:
          排除

      momentum_rank = rank(momentum, descending)
      premium_rank = rank(premium, ascending)
      score = momentum_weight × momentum_rank + premium_weight × premium_rank

  选取score最优的前hold_count只 → 等权买入
```

**可配置参数**：

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| etf_pool | 10只行业ETF | 用户选择 | 轮动标的池 |
| hold_count | 3 | 1~10 | 持仓数量 |
| rebalance_day | friday | 周几 | 调仓日 |
| lookback_days | 20 | 5~60 | 动量回看天数 |
| momentum_weight | 0.7 | 0~1 | 动量因子权重 |
| premium_weight | 0.3 | 0~1 | 溢价率因子权重 |
| momentum_threshold | 0 | -0.1~0.05 | 最低动量门槛 |
| min_amount | 1000万 | 0~1亿 | 最低日均成交额 |
| max_same_sector | 2 | 1~5 | 同行业最大持仓数 |
| drawdown_stop | 0.15 | 0.05~0.30 | 个股回撤止损阈值 |

### 回测参考
- 果仁网回测：年化约37%（14只标的，取前10名，每周轮动）
- 添加成交额过滤后收益略降但稳定性提升

## B7. 果仁低估值轮动

### 来源
果仁网，以银行ETF为案例的估值因子策略。

### 原版逻辑

```
行业范围：银行（或其他单一行业）

每个调仓日（每15个交易日）：
  FOR 每只个股/ETF in 行业:
      pe_rank  = 按PE升序排名（PE越低越好）
      pb_rank  = 按PB升序排名（PB越低越好）
      div_rank = 按股息率降序排名（股息越高越好）

      composite_rank = (pe_rank + pb_rank + div_rank) / 3

  选取composite_rank排名第1的标的 → 全仓买入
```

### 核心特点
- **纯估值驱动**：PE + PB + 股息率三因子
- **单一行业内轮动**：不跨行业，降低行业风险
- **低频调仓**：15个交易日一次
- **集中持仓**：只持有1只（原版），可调整

### 我们的复刻方案（泛化为跨行业版）

将单一行业内的估值轮动扩展为跨行业ETF估值轮动：

```
ETF池 = 宽基/行业ETF列表

每rebalance_period个交易日：
  FOR etf in pool:
      pe_percentile = 当前PE在历史N年PE中的百分位
      pb_percentile = 当前PB在历史N年PB中的百分位
      dividend_yield = 最近12个月股息率

      # 估值评分：百分位越低越便宜
      value_score = (1 - pe_percentile) × pe_weight
                  + (1 - pb_percentile) × pb_weight
                  + dividend_yield × div_weight

  选取value_score最高的前hold_count只 → 等权买入
```

**可配置参数**：

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| etf_pool | 宽基+行业ETF | 用户选择 | 标的池 |
| hold_count | 3 | 1~5 | 持仓数量 |
| rebalance_period | 15 | 5~60 | 调仓周期（交易日） |
| pe_weight | 0.4 | 0~1 | PE因子权重 |
| pb_weight | 0.3 | 0~1 | PB因子权重 |
| div_weight | 0.3 | 0~1 | 股息率因子权重 |
| history_years | 5 | 3~10 | 百分位计算的历史区间 |

### 回测参考
- 银行ETF低估值轮动：年化17.57%，最大回撤29.07%（2014-2024）
- 对比基准沪深300显著跑赢

## B8. 华宝网格交易

### 来源
华宝智投APP，ETF网格交易机器人。

### 原版逻辑

```
初始化：
  选定target_etf
  设定价格区间 [price_lower, price_upper]
  设定网格数量 grid_count
  计算网格间距 grid_step = (price_upper - price_lower) / grid_count
  确定基准价 base_price = 当前价格
  建立底仓

每个交易日（盘中逐K线判断）：
  current_price = 当前价格

  # 等差网格
  IF current_price <= base_price - grid_step:
      买入一格（金额 = order_amount）
      base_price 下移一格
  ELIF current_price >= base_price + grid_step:
      卖出一格
      base_price 上移一格

  # 仓位保护
  IF 总持仓 >= max_position:
      不再买入
  IF 总持仓 <= min_position:
      不再卖出

  # 价格越界
  IF current_price > price_upper OR current_price < price_lower:
      网格休眠，不操作
```

### 华宝智投的关键参数设计

| 参数 | 华宝实现 | 说明 |
|------|----------|------|

### 我们的复刻方案

```
grid_type: arithmetic (等差) 或 geometric (等比)

等差网格：
  grid_step = (price_upper - price_lower) / grid_count
  第i格价格 = price_lower + i × grid_step

等比网格：
  ratio = (price_upper / price_lower) ^ (1 / grid_count)
  第i格价格 = price_lower × ratio^i

回测模拟（日K线级别）：
  每日检查 high 和 low 是否穿越任何网格线
  IF low <= 某格买入线:
      模拟买入（按该格价格成交）
  IF high >= 某格卖出线:
      模拟卖出（按该格价格成交）
```

| total_budget | 100000 | > 0 | 总投入资金 |
| use_rebound_confirm | false | true/false | 回落确认（触格后等回落再成交） |
| min_position_grids | 2 | 0~5 | 最小底仓（格数） |
| max_position_pct | 0.8 | 0.5~1.0 | 最大仓位占比 |

## B9. 集思录动量+RSRS择时

### 来源
集思录 + 知乎量化社区，10年17倍策略的核心思路。

### 原版逻辑

```
═══ 第一层：动量评分（决定买什么）═══

ETF池 = [黄金ETF, 纳指ETF, 创业板ETF, 沪深300ETF, 国债ETF]

FOR etf in pool:
    # 计算对数收益率的线性回归斜率
    log_returns = log(close / close.shift(1))  # 过去25天
    slope = linear_regression_slope(log_returns, days=25)
    annualized_return = slope × 252
    r_squared = linear_regression_r2(log_returns, days=25)

    # 短期动量得分
    momentum_score = annualized_return × r_squared

    # 长期反转修正（200天）
    long_return = log(close / close_200d_ago)
    reversal_score = -long_return  # 符号取反，捕捉反转

    # 综合得分
    final_score = momentum_score + reversal_weight × reversal_score

选取final_score最高的ETF → 全仓买入


═══ 第二层：RSRS择时（决定是否入场）═══

FOR 持仓ETF:
    # 计算RSRS指标
    high_series = 过去18天最高价
    low_series  = 过去18天最低价
    slope = OLS回归(因变量=high, 自变量=low).斜率

    # 标准化（过去600天的斜率序列）
    z_score = (slope - mean_250d) / std_250d
    r2 = OLS回归的决定系数
    rsrs = z_score × r2

    IF rsrs > rsrs_threshold:
        确认持有（动量+择时双确认）
    ELSE:
        卖出 → 转入货币ETF


═══ 组合逻辑 ═══

每日收盘后：
  1. 计算所有ETF的动量得分
  2. 选出得分最高的ETF
  3. 用RSRS检验该ETF是否处于强势
  4. 若RSRS确认 → 持有；若RSRS否决 → 持有货币ETF
```

## B10. ETF组合宝行业轮动

### 来源
ETF组合宝APP（etf.group），"十大AI推荐行业"功能。

### 逆向推测逻辑

ETF组合宝未公开完整算法，根据公开信息和行业通行做法推测：

```
═══ 行业评分模型（多因子）═══

行业ETF池 = 全市场行业/主题ETF

FOR etf in pool:
    # 因子1：动量（权重最高）
    momentum_10d = 近10日涨幅
    momentum_20d = 近20日涨幅

    # 因子2：成交量变化
    volume_ratio = 近5日均量 / 近20日均量  # 放量程度

    # 因子3：资金流向（如有数据）
    fund_flow = 近5日净流入金额

    # 因子4：波动率（低波优先或高波优先可选）
    volatility = 近20日收益率标准差

    # 综合评分
    score = w1 × rank(momentum_20d)
          + w2 × rank(volume_ratio)
          + w3 × rank(fund_flow)
          + w4 × rank(-volatility)  # 低波加分

选取score前10名 → 作为"十大推荐行业"
前3名 → 模拟交易买入标的
```

### 核心特点
- **AI标签**：可能使用机器学习对因子权重做动态优化
- **每日更新**：十大推荐列表每个交易日更新
- **模拟交易验证**：用户可跟随模拟交易，自定义买入时间
- **灵活跟车**：不需要开盘就买，任意时间点效果差不多
